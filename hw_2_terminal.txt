Для выполнения задания у вас должен быть установлен для Windows - GitBash.
Создан аккаунт в GitHub

Все шаги сценария выполняйте в терминале GitBush, Terminal, в папке под гитом.

Как отправить ДЗ на проверку.
 1. Создайте текстовый файл как в первом ДЗ по Terminal.
 2. Сценарий перенесите в этот файл.
 3. На против каждого действия - напишите команду в GitBash

Файл со сценарием и ссылку на свой гит хаб отправляйте менторам на проверку.

JSON
 4. Создать внешний репозиторий c названием JSON - через польз. веб-интерфейс, new
 5. Клонировать репозиторий JSON на локальный компьютер - git clone *ссылка на репозиторий*
 6. Внутри локального JSON создать файл “new.json”. - cat > new.json
 7. Добавить файл под гит. - git add new.json
 8. Закоммитить файл. - git commit -m "first comm" new.json (после можно исп. git commit -am "other comm", где а - заменит git add ., но это после 1 коммита)
 9. Отправить файл на внешний GitHub репозиторий. - git push
 10. Отредактировать содержание файла “new.json” - написать информацию о себе (ФИО, возраст, количество домашних животных, будущая желаемая зарплата). Всё написать в формате JSON. - через vim new.json (заполнять под cat - криво)
 11. Отправить изменения на внешний репозиторий - git commit -m "second comm" new.json, git push
 12. Создать файл preferences.json - cat > preferences.json
 13. В файл preferences.json добавить информацию о своих предпочтениях (Любимый фильм, любимый сериал, любимая еда, любимое время года, сторона которую хотели бы посетить) в формате JSON. - через vim preferences.json
 14. Создать файл sklls.json добавить информацию о скиллах которые будут изучены на курсе в формате JSON - bug (назовем файл skills.json): cat > skills.json, vim skills.json
 15. Отправить сразу 2 файла на внешний репозиторий. -  git add preferences.json skills.json, git commit -m "4 commit" preferences.json skills.json, git push
 16. На веб интерфейсе создать файл bug_report.json. - через кнопку add file в репозитории, примечание: числа - без кавычек - только в значении (а не в ключе)
 17. Сделать Commit changes (сохранить) изменения на веб интерфейсе. - через кнопку commit new file
 18. На веб интерфейсе модифицировать файл bug_report.json, добавить баг репорт в формате JSON. - нажать на кнопку Edit this file
 19. Сделать Commit changes (сохранить) изменения на веб интерфейсе - нажать на кнопку Commit changes
 20. Синхронизировать внешний и локальный репозиторий JSON - git pull


XML
 21. Создать внешний репозиторий c названием XML. - СОЗДАДИМ ЧЕРЕЗ BASH (Получить токен на github, mkdir XML, cd XML, git init, echo "This repo was created remotely" >> README.md (по желанию), git add ., git commit -m "first commit", git remote add origin https://github.com/USER/XML.git (ссылка придумывается), curl -u 'USER:TOKEN' https://api.github.com/user/repos -d '{"name":"XML"}', git push --set-upstream origin master
USER - логин на гитхабе
TOKEN - токен типа 6ccfd64d55fc1ca1cc26ffe2b9351cc9)
 22. Клонировать репозиторий XML на локальный компьютер - git clone *ссылка на репозиторий* - уже локально, т. к. создавали локально
 23. Внутри локального XML создать файл “new.xml”. - touch new.xml
 24. Добавить файл под гит. - git add .
 25. Закоммитить файл. - git commit -m "second commit"
 26. Отправить файл на внешний GitHub репозиторий. - git push
 27. Отредактировать содержание файла “new.xml” - написать информацию о себе (ФИО, возраст, количество домашних животных, будущая желаемая зарплата). Всё написать в формате XML. - vim new.xml
 28. Отправить изменения на внешний репозиторий. - git commit m "third commit", git push
 29. Создать файл preferences.xml - touch preferences.xml        примечание: можно использовать конвертер из json :-)
 30. В файл preferences.xml добавить информацию о своих предпочтениях (Любимый фильм, любимый сериал, любимая еда, любимое время года, сторона которую хотели бы посетить) в формате XML. vim preferences.xml
 31. Создать файл sklls.xml (мб skills.xml) добавить информацию о скиллах которые будут изучены на курсе в формате XML - touch skills.xml && vim skills.xml
 32. Сделать коммит в одну строку. - git add . && git commit -m "fourth commit"
 33. Отправить сразу 2 файла на внешний репозиторий. - git push
 34. На веб интерфейсе создать файл bug_report.xml. - через кнопку add file в репозитории, примечание: В тегах не должно быть пробелов, если это не атрибут, теги не должны начинаться с цифр
 35. Сделать Commit changes (сохранить) изменения на веб интерфейсе. - через кнопку commit new file
 36. На веб интерфейсе модифицировать файл bug_report.xml, добавить баг репорт в формате XML. - нажать на кнопку Edit this file
 37. Сделать Commit changes (сохранить) изменения на веб интерфейсе. - нажать на кнопку Commit changes
 38. Синхронизировать внешний и локальный репозиторий XML - git pull
